# ClawTeam Qwen Swarm Orchestrator 🦞🚀

โปรเจกต์นี้เป็นการนำเฟรมเวิร์ก **ClawTeam (v0.2.0)** มาใช้ควบคุมฝูงเอเจนต์ (Swarm) โดยใช้ **Qwen Code (qwencode)** เป็นเครื่องยนต์หลักในการทำงาน เพื่อแก้ปัญหาการติดยืนยัน (Security Approval) ใน OpenClaw รุ่นใหม่ๆ

## 🌟 ที่มาที่ไป (Background)
เนื่องจาก OpenClaw เวอร์ชันปัจจุบันมีการเข้มงวดเรื่องความปลอดภัย (ACP) ทำให้เวลาสั่งรันผ่าน Terminal มักจะหยุดรอการอนุมัติ (Approve) ตลอดเวลา เราจึงแก้ปัญหาด้วยการ:
1. ใช้ **Qwen Code** ที่รองรับ Flag `-y` หรือ `--yolo` เพื่อข้ามการขออนุญาตโดยตรง
2. ใช้ **ClawTeam** เป็นตัวควบคุม (Orchestrator) เพื่อให้เอเจนต์ 5 ตัว (Code-Arch, Sec-Rev, Logic-Rev, Perf-Anal, Code-Qual) คุยกันเองและส่งงานกันได้เป็นระบบ
3. รันผ่าน **Tmux Backend** เพื่อให้เราสามารถส่องดูการทำงานแบบ Real-time ได้ในหน้าจอเดียว

---

## 🚀 วิธีการเรียกใช้งาน (Getting Started)

1. **เตรียมความพร้อม:**
   ตรวจสอบว่าไฟล์ `claude` (Symbolic link ไปสู่ qwen) และ `launch_team_qwen_interactive.sh` พร้อมทำงาน

2. **สั่งรันทีม:**
   รันสคริปต์หลักเพื่อเริ่มสร้างทีมและแจกจ่ายงาน:
   ```bash
   ./launch_team_qwen_interactive.sh
   ```

3. **ผลลัพธ์ของงาน:**
   เอเจนต์จะถูกบังคับให้เขียนไฟล์ลงในโฟลเดอร์ **`app/`** เท่านั้น (เช่น `app/api.py`, `app/test_api.py`) เพื่อความเป็นระเบียบ

---

## 📊 การติดตามการทำงาน (Monitoring)

### 1. ดูผ่านหน้าเว็บ Dashboard
สามารถดูสถานะของ Task ทั้งหมด, ข้อความที่เอเจนต์คุยกัน และความคืบหน้าภาพรวมได้ที่:
👉 [http://localhost:8081/](http://localhost:8081/)

### 2. ดูผ่านหน้าจอ Tmux (Terminal)
หากต้องการส่องดูหน้าจอ Command Line ของเอเจนต์แต่ละตัวที่กำลังรันอยู่แบบ Real-time:
```bash
clawteam board attach qwen-team
```
*หมายเหตุ: หากหน้าจอไม่หั่นเป็น 4 ส่วนโดยอัตโนมัติ ให้ลองกด `Ctrl+b` แล้วตามด้วยเลข `0` เพื่อไปที่หน้าหลักที่รวมทุกเอเจนต์ไว้*

---

## 🛠️ โครงสร้างโฟลเดอร์
- `app/`: โฟลเดอร์ที่เก็บเฉพาะผลงานที่เอเจนต์สร้างขึ้น
- `archive/`: โฟลเดอร์เก็บสคริปต์เก่าและไฟล์ที่ไม่ได้ใช้งานแล้ว
- `launch_team_qwen_interactive.sh`: สคริปต์หลักในการตั้งทีมและสั่งรัน
- `claude`: Symbolic Link (Mask) ที่หลอก ClawTeam ให้ทำงานในโหมด Interactive ได้
