# Antigravity Context Handover
**Date:** 2026-04-04
**Project:** OpenClaw Auth Lab (10-Agent Full SDLC Swarm)

เอกสารนี้ใช้สำหรับทบทวนและแจ้งให้ Agent (Antigravity) ใน Session ต่อไปทราบถึงบริบทและสิ่งที่เราได้ทำไปล่าสุด เพื่อให้สามารถสานต่อการทำงานได้อย่างไร้รอยต่อ

---

## 1. บริบทและเป้าหมายล่าสุด (Current Goal)
เรากำลังเซ็ตอัประบบโครงสร้าง **10-Agent Autonomous SDLC Swarm** ให้กับโปรเจกต์ `openclaw_auth_lab` โดยใช้ ClawTeam framework ควบคู่ไปกับฟีเจอร์ของ OpenClaw 

## 2. การปรับปรุงสถาปัตยกรรม (Architecture Changes)
เราได้ทำการยกเครื่องระบบ **Workspace Isolation** ใหม่ทั้งหมด:
- **ปัญหาก่อนหน้า:** การระบุ `--workspace` ไว้ที่โฟลเดอร์ผลลัพธ์ (`app/`) ทำให้โครงสร้างโปรเจกต์รก เพราะระบบพ่นไฟล์จัดการความจำตัวเอง (`.openclaw`, `AGENTS.md`, ฯลฯ) ลงมาปนกับ Source Code
- **การแก้ไข (Agent Sandbox Isolation):** 
  - สร้างถังขยะความจำคู่ขนาน: ขังเอเจนต์ไว้ที่ `agent_sandbox/` ตลอดชีพ 
  - โฟลเดอร์ `app/` จะมีไว้เพื่อรองรับ Source Code บริสุทธิ์อย่างเดียว โดยปรับ Prompt ของทุก Agent ใน `launch.sh` ว่าให้เซฟงานทุกชิ้นไปที่ `../app/` แทน
- ลบสคริปต์ `setup_workspace_meta.sh` ออกและเลิกใช้ระบบ Symlink โบราณทั้งหมด
- อัปเดตคู่มือหลักใน `openclaw_ws_setup.md` ให้สะท้อนการเปลี่ยนแปลงนี้เรียบร้อยแล้ว

## 3. แก้บั๊กระดับวิกฤต (Critical Bug Fixes)
- **สายพานงานพัง (Parallel Run Bug):** เราพบว่าฟังก์ชัน Task Blocking พัง เนื่องจากสคริปต์ไปใช้ `grep -o 'id: [a-f0-9]*'` จับหา Task ID แต่ข้อความจาก ClawTeam เปลี่ยนฟอร์แมต ทำให้ได้ค่าว่าง ($T0-$T9 ว่างเปล่า) ส่งผลให้ออกสตาร์ทพร้อมกันหมดและแย่งงานกันพัง
- **การแก้ไข:** เปลี่ยนไปใช้ท่า `grep 'Task created:' | awk '{print $4}'` ดึงรหัสงานใน `launch.sh` เรียบร้อย ตอนนี้สายงานเป็นลำดับขั้น Sequential อย่างสมบูรณ์แบบ

## 4. จัดเตรียมเครื่องมือ (New Tooling)
- เพิ่มสคริปต์ **`clean.sh`** เพื่อความสะดวกในการรีเซ็ตแล็บ โดยมันจะรับหน้าที่เคลียร์แคช ClawTeam ทั้งหมด, ลบ `$SCRIPT_DIR/app` ลบ `agent_sandbox` และ Kill Tmux sessions เก่าๆ ทิ้งแบบ 100%

## 5. การมอนิเตอร์และข้อควรระวัง (Monitoring & Dashboards)
- ทีม `openclaw-team` โฉมใหม่ประกอบด้วยตัวละครหลัก 10 ทหารเสือ
- **พอร์ต 8080 (Avatar Dashboard):** หน้าจอศิลปะพิกเซลนี้ "ใช้ไม่ได้" สำหรับโปรเจกต์นี้ เพราะมันถูกออกแบบหน้าตา UI มาเพื่อทีม `qwen-team` รุ่นบุกเบิกที่มีแค่ 6 โต๊ะทำงาน 
- **พอร์ต 8081 (ClawTeam Board):** เราต้องใช้หน้าจอนี้เป็นหลักในการสังเกตการณ์สถานะ Pending, In Progress, Completed เพราะรองรับครบ 10 ตัวละคร

## 6. สิ่งที่ต้องทำต่อไป (Next Steps)
พื้นแพเทคนิคและโค้ดทั้งหมด (รวม `summary_script.txt`) ถูก Commit ลง Git เรียบร้อย ในเซสชั่นถัดไป สิ่งที่เราควรทำคือ:
1. เรียกใช้ `./clean.sh` (เผื่อชัวร์)
2. สั่งรัน `./launch.sh`
3. เฝ้าดูประสิทธิภาพของ Swarm ทั้ง 10 ตัว ผ่านหน้า Dashboard 8081 ว่าทำงานสอดประสานกันตั้งแต่ System Planning ไปจนถึง Project Management แบบไร้รอยต่อได้จริงหรือไม่!
