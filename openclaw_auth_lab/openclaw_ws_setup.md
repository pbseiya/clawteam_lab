# 🦞 OpenClaw: Agent Workspace Isolation Guide

เอกสารฉบับนี้อธิบายวิธีการตั้งค่าเพื่อป้องกันไม่ให้ OpenClaw เอเจนต์บันทึกไฟล์ลงในที่อยู่ที่ทำงานเริ่มต้น (`~/projects/openclaw/workspace`) และบังคับให้มาทำงานในโฟลเดอร์ของโปรเจกต์โดยตรง (Project Isolation)

---

## 🔍 ปัญหาที่พบ (The Problem)

โดยปกติ OpenClaw เอเจนต์จะมี **"บ้านถาวร" (Persistent Workspace)** ของตัวเอง ซึ่งถูกกำหนดไว้ในคอนฟิกตอนสร้าง (Bootstrap) เมื่อเราสั่งให้เอเจนต์ทำงานผ่าน `clawteam` หรือ `openclaw` ต่อให้เราสั่งจากในโฟลเดอร์โปรเจกต์ เอเจนต์ก็จะบันทึกไฟล์ลงในบ้านของมันเสมอ ทำให้เกิดปัญหาดังนี้:
1. **งานปนกัน (Cross-contamination):** ผลลัพธ์จากหลายโปรเจกต์ไปรวมกันอยู่ที่เดียว
2. **ความไม่เป็นระเบียบ:** ไฟล์งานไม่ได้ถูกเก็บไว้ในที่ที่มันควรอยู่ (ร่วมกับ Source Code ของโปรเจกต์นั้นๆ)
3. **การจัดการ Git ยาก:** ผลงานเอเจนต์ไม่ถูกนับรวมเข้าไปใน Repository ของแล็บ

---

## ✅ วิธีการแก้ไขที่ถูกต้อง (Official Solution)

วิธีที่ดีที่สุดคือการใช้ระบบ **`OPENCLAW_PROFILE`** ร่วมกับคำสั่ง **`openclaw setup --workspace`** เพื่อสร้าง "Sandbox" สำหรับโปรเจกต์นั้นๆ โดยเฉพาะครับ

### 🛠️ ขั้นตอนการตั้งค่าในสคริปต์ (`launch.sh`)

```bash
# 1. ตั้งค่า Profile Name (เปลี่ยนตามชื่อโปรเจกต์)
export OPENCLAW_PROFILE=openclaw_auth_lab

# 2. ตั้งค่า Workspace ให้มาที่โฟลเดอร์ที่ต้องการ (เช่น โฟลเดอร์ app/ ในโปรเจกต์)
openclaw setup --workspace "$(pwd)/app"

# 3. (ทางเลือก) หากต้องการให้ Named Agents ตนเดิมรับทราบ Workspace ใหม่
openclaw agents add code-arch --workspace "$(pwd)/app"
openclaw agents add sec-rev   --workspace "$(pwd)/app"
# ... ทำซ้ำสำหรับเอเจนต์ทุกตัวในทีม ...
```

## 📂 เหตุผลเบื้องหลัง (Rationale)

### 1. Project-Based Isolation
การใช้โปรไฟล์ทำให้เอเจนต์มี "ความจำและที่ทำงาน" แยกกันตามชื่อโปรไฟล์นั้นๆ เมื่อคุณสลับไปทำงานโปรเจกต์อื่น (โดยไม่เปิดโปรไฟล์นี้) เอเจนต์ก็จะกลับบ้านเดิม (`default`) ตามปกติ ไม่มากวนงานเดิมของคุณครับ

### 2. Clean Global Config
แทนที่จะไปแก้ไขพาธในคอนฟิกถาวรของเอเจนต์ทุกตัว (ซึ่งอาจจะพัง หรือลืมแก้คืน) เราใช้การ "Override" เฉพาะในบริบท (Context) ของโปรเจกต์นี้เท่านั้น ทำให้คอนฟิกหลักของคุณยังคงความสะอาดและเป็นมาตรฐานครับ

## 🔒 ความปลอดภัยและการใช้งานข้ามโปรเจกต์ (Scope & Safety)

มีข้อสงสัยว่า **"หากใช้คำสั่ง `export` แล้ว เมื่อไปทำโปรเจกต์อื่น เอเจนต์จะยังมาสร้างไฟล์ในที่เดิมนี้อีกไหม?"**

### คำตอบคือ: **"ไม่ครับ!"** 🚫📁

เหตุผลด้านเทคนิค (Technical Reasons):
1. **Script Scope (ขอบเขตของสคริปต์):** เมื่อเราใส่ `export OPENCLAW_PROFILE=...` ไว้ในไฟล์ `launch.sh` ตัวแปรนี้จะ **มีผลเฉพาะภายในสคริปต์นั้นและกระบวนการลูก (เอเจนต์)** ที่สคริปต์สั่งรันเท่านั้น เมื่อสคริปต์ทำงานเสร็จหรือสคริปต์หยุดทำงาน ตัวแปรนี้จะ "ทำลายตัวเอง" ทันทีครับ
2. **Terminal Isolation (การแยกส่วนของเทอร์มินัล):** หากคุณเปิด Terminal ใหม่หรือ SSH เข้ามาใหม่โดยไม่รันสคริปต์นี้ ค่า `OPENCLAW_PROFILE` จะกลับไปเป็นค่าเริ่มต้น (หรือเป็น `default`) ทำให้เอเจนต์กลับไปใช้ Workspace ปกติของเขาตามปกติครับ
3. **No Global Contamination:** วิธีนี้ **ไม่แก้ไขตัวแปรระบบ (System Environment)** ของผู้ใช้ภายนอกสคริปต์ ดังนั้นจึงรับประกันได้ว่าโครงการอื่นๆ จะไม่ถูกรบกวนแน่นอนครับ

---

> [!IMPORTANT]
> **ข้อควรระวัง:** การใช้โปรไฟล์แยกจะทำให้เอเจนต์ **"จำไม่ได้"** ว่าเคยคุยอะไรกับคุณในโปรไฟล์อื่น (Session Isolation) ซึ่งถือเป็นเรื่องดีสำหรับการแยกโปรเจกต์ให้ขาดจากกันครับ

> [!TIP]
> หากต้องการลบโปรไฟล์นี้ทิ้งในภายหลัง (Prune) สามารถใช้คำสั่ง:
> `rm -rf ~/.openclaw/agents/*/sessions/<profile_name>`
